How to install RobOptim on your machine?

Choose your Linux Distribution in the list or choose source install if your distribution is unsupported. Instructions are for the Core package but other packages can be installed following the same steps and replace "core" by the appropiate package you want to install.

  • Arch Linux
  • Debian / Ubuntu
  • Fedora
  • robotpkg
  • Source

While not available in the official repositories, RobOptim is available on Arch Linux using AUR.

If you are using Arch Linux, please open a terminal and run the following command:

{% highlight bash %} # To install RobOptim Core with yaourt: $ yaourt roboptim-core # or with packer: $ packer roboptim-core {% endhighlight %}

If you are using Debian or Ubuntu, please open a terminal and run the following command:

{% highlight bash %} $ sudo apt-get install libroboptim-core-dev {% endhighlight %}

If you are using Fedora, please open a terminal and run the following command:

{% highlight bash %} $ sudo yum install roboptim-core-devel {% endhighlight %}

If you are using RobotPkg, please open a terminal and run the following command:

{% highlight bash %} # from your robotpkg directory $ cd optimization/roboptim-core && make update {% endhighlight %}

Compiling from source is significantly more challenging than installing a binary package. Be prepared to face compilation errors, software bugs and crashes.

Dependencies

First, install the following tools on your system:
  • CMake
  • gcc, g++
  • Doxygen (and eventually LaTeX for mathematical formula rendering)
  • Boost
  • Eigen
  • Log4cxx
And then follow one of the two next steps depending on your needs.

Installing a release from source

{% highlight bash %} # Uncompress the tarball (replace X.Y by the proper version number) $ tar xzvf roboptim-core-X.Y.tar.gz # Move to the newly created directory $ cd roboptim-core-X.Y # Create build directory $ make _build && cd _build # Run cmake or ccmake and set the installation options as needed # Note: RelWithDebInfo is the prefered value for CMAKE_BUILD_TYPE # Debug (or no value) is considerable slower than release mode. $ cmake -DCMAKE_BUILD_TYPE=RelWithDebInfo -DCMAKE_INSTALL_PREFIX=/your/install/prefix .. # Build the software $ make # Run the test suite $ make test # Install the software $ make install {% endhighlight %}

Installing from Git

{% highlight bash %} $ git clone --recursive git://github.com/roboptim/roboptim-core.git # Move to the newly created directory $ cd roboptim-core # Create build directory $ make _build && cd _build # Run cmake or ccmake and set the installation options as needed # Note: RelWithDebInfo is the prefered value for CMAKE_BUILD_TYPE # Debug (or no value) is considerable slower than release mode. $ cmake -DCMAKE_BUILD_TYPE=RelWithDebInfo -DCMAKE_INSTALL_PREFIX=/your/install/prefix .. # Build the software $ make # Run the test suite $ make test # Install the software $ make install {% endhighlight %}
